How does a buyer actually go act on the deal?

Finding the address, the contact, and the way to reach them — legally.
Prepared by Scout · 2026-06-14 · For Atlas (build) with a note for Serena (positioning)

The one-paragraph answer

Good news: we already have what we need, and it's legal. Our existing source RentCast returns the full property address AND a listingAgent and listingOffice block — each with name, phone, email, and website. RentCast's API terms explicitly allow us to display this data to our end users in a paid app, with no attribution required. So the simplest compliant path is: show the agent's name + phone + email right on the deal detail page, with a "Call / Email the listing agent to see this home" button. No scraping, no new account, no new spend. We are not currently mapping those fields — that's the only gap, and it's a small build for Atlas.

1. What contact & address data is legally available to us

RentCast (our existing source) — RECOMMEND · already paid for

This is the headline finding. The RentCast sale listings response includes:

FieldWhat it gives the buyer
formattedAddressThe full address: Street, Unit, City, State Zip. The buyer can find and drive to the home.
listingAgent → name, phone, email, websiteThe person to call. The actual listing agent representing the seller.
listingOffice → name, phone, email, websiteThe brokerage/office — a fallback contact if the agent line is blank.
mlsName, mlsNumberThe MLS listing ID — lets a buyer (or their own agent) look the home up officially.
listingTypeStandard / New Construction / Foreclosure / Short Sale — tells the buyer what kind of sale it is.
Why this matters in plain terms: a new data field here is like a phone number printed right on the price tag. The buyer sees the deal, sees who's selling it, and can ring them. We don't have to send them off to Zillow to figure it out.

Honest caveat (unverified): RentCast pulls from MLS feeds, so the agent/office block is present on most for-sale listings but probably not 100% — some records may have a blank agent. Plan for that: fall back to listingOffice, then to a "find a local agent" link. We can only confirm the real fill-rate once the worker pulls live data (RentCast is quota-frozen until July 7 — don't run it before then).

Foreclosure / bank-owned (REO) homes — MAYBE · weak on contact

For bank-owned homes the contact question is different. Our (dormant, not-subscribed) Foreclosure Data Hub source returns address, owner name, county, sale/auction date, bid amount, a Redfin value estimate, and a direct link to the property — but its public API docs do not clearly expose an REO agent, bank, or trustee contact.

  • How REO actually works: the bank assigns the property to a local REO listing agent, and you contact that agent (or your own buyer's agent) — you do not cold-call the bank. (Source: RealtyTrac support.)
  • So for foreclosures, the cleanest contact is still the listing agent — which means if/when a foreclosure also appears as an MLS listing, RentCast's listingAgent is again the answer.
  • Foreclosure Data Hub is $49/mo and we are not subscribed (trial expired). I do not recommend paying just to chase a contact field its docs don't even promise. Owner decision if ever wanted — but skip for now.

Other "how to contact" sources — SKIP / not needed

SourceVerdictWhy
Zillow / Redfin / Realtor.com "contact agent" widgetsSKIP — legal riskScraping their pages or embedding their contact flow breaks their terms. Hard no. (This is the guardrail.)
ATTOM / public county recordsSKIP for contactGive owner name, not the selling agent. Contacting a homeowner who listed with an agent is poor form and edges toward solicitation. Not the buyer's path.
"Find an agent" referral networksMAYBE (later)Useful as a fallback when a listing has no agent — and a possible revenue idea for Serena (referral fees). Not needed for v1.

2. What are we ALLOWED to show? (licensing & Fair Housing)

RentCast API Terms — the green lights:
  • ✅ Display to end users of a commercial app is permitted ("display, resale and distribution of the API Data to third parties").
  • ✅ Store it in our cache ("incorporate and store the API Data within your internal systems") — fits our shared-cache model perfectly.
  • ✅ No attribution required ("neither use of the RentCast API nor the API Data requires attribution or disclosure").
The lines we must NOT cross (RentCast Terms):
  • 🚫 No automated outreach. We can show the agent's phone/email so the buyer chooses to contact them. We must not auto-email or bulk-message agents from our system. ("must not send any unsolicited commercial e-mail" / "automated queries".)
  • 🚫 No using the data for marketing/solicitation in any state that bans using public info that way. Translation: this is a buyer-facing "here's who to call" feature, not an agent-marketing list.
  • 🚫 Don't let third parties scrape the data back out of our app.
Safe design rule: the buyer taps a button → it opens their own phone or email app pre-filled (a tel: or mailto: link). We never send the message; the buyer does. That keeps us clearly on the right side of the "no automated solicitation" line.
⚖️ Fair Housing note (loud, on purpose): showing an address and a listing-agent contact is fine — it's neutral transaction info. The rule we never break is using crime / demographic / "neighborhood quality" / "school" data to score or filter homes. Nothing in this feature touches that. One thing to watch: any pre-filled message template must be plain ("I'd like to see this home") and must never let a buyer filter or ask about the people in an area. Keep templates about the property only.

3. How competitors & normal real-estate sites do it

Across Zillow, Trulia, StreetEasy, Redfin and agent sites, there are basically four patterns, from heaviest to lightest:

PatternWhat it isFit for our beginner / live-in buyer
"Request a Tour"Buyer fills a form; platform routes it to an agent (often a paid "premier"/referral agent, not always the listing agent).Strongest CTA for ready buyers — but needs a backend + an agent network we don't have. Overkill for v1.
"Contact Agent" formA message box that emails the listing agent.Good, but building/sending email on the buyer's behalf invites the "automated outreach" risk. Avoid sending it ourselves.
Show agent + phone/email directlyDisplay the listing agent's name and number; buyer calls/emails themselves.Best fit. Simplest, cheapest, zero backend, clearly compliant. This is exactly what RentCast lets us show.
Link out to the public listingA button to the home's MLS/listing page.Useful as a secondary "see full listing" link — but never a scraped portal. Use the website / MLS number we're licensed for.

Research note: industry guidance says "Request a Tour" converts best for bottom-of-funnel buyers, but it assumes an agent-routing backend. For a beginner-simple v1 with no such backend, showing the real contact + a one-tap call/email delivers 90% of the value at ~10% of the build.

4. Plain recommendation — the cheapest compliant path

On the deal detail page, under the score, add a small "Go see this home" block:
  1. Address (we already get formattedAddress).
  2. Listing agent: name + brokerage (from listingAgent / listingOffice).
  3. Two buttons: 📞 Call (a tel: link) and ✉️ Email (a mailto: link pre-filled with a plain, property-only message). The buyer's own phone/email app sends it — never ours.
  4. Fallbacks when the agent block is blank: show the office contact; if that's also blank, show a neutral "Ask a local agent to see this home (MLS #...)" line.
Cost: $0. No new source, no signup, no extra RentCast calls (the agent fields ride along in the listing data we already pull and cache).
