Go-Live Checklist — Worker + Payment

Closing the gate so the app shows real homes and can take payment · 2026-06-13

The code is finished. The refresh worker, its daily schedule, and the Payhip "Subscribe" button are all built and committed. Everything below is account / dashboard steps — no coding. Do them in order.

Part A — Turn the worker ON (fills the app with real homes)

⏳ Wait until July 7 (or upgrade RentCast first). Our free RentCast quota is used up for this billing period (resets July 7, 2026). The worker's first run makes RentCast calls — so don't turn it on before the reset, or you may hit overage. Either wait until July 7, or upgrade RentCast (Foundation $74/mo) first. Customers are unaffected meanwhile (the app reads cached data).

1 Add the secrets to GitHub

Go to your repo → Settings → Secrets and variables → Actions → New repository secret. Add each of these (name on the left, your real value on the right):

DATABASE_URL  =  (the same Supabase string the app uses)
RENTCAST_API_KEY  =  (your RentCast key)
RESEND_API_KEY  =  (optional — for alert emails)
ALERT_FROM_EMAIL  =  (optional — the "from" address)

These live ONLY in GitHub's secret box — never typed into a file. The worker reads them at run time.

2 Run it once by hand

Repo → Actions tab → "Refresh listings cache" → Run workflow. Watch it go green (takes a few minutes). After that it runs automatically every day at ~9:00 UTC.

3 Confirm real homes appeared

Open the live app → it should now show real listings. Or open Admin / Usage → backend should read PostgreSQL and the listing count should be above zero.

Heads-up on coverage: right now the worker only pulls 6 Northern California cities (Sacramento, Stockton, Modesto, Vallejo, Antioch, Oakland). So the app will show real homes there. To cover more of the USA we need a small code tweak (the city list currently allows only one state) — see the decision at the bottom.

Part B — Turn payment ON ($12.99/mo)

1 Create the product on Payhip

Sign up at payhip.com → create a Subscription product → price it $12.99/month (you can add the "$12.99 for 12 months, then $29.99" framing in the description). Payhip handles the card payment and gives you a link like https://payhip.com/b/XXXXX.

2 Paste the link into one file

Open config/cache.yaml, line 15, and put your link between the quotes:

checkout_url: "https://payhip.com/b/XXXXX"

Save, commit, push. The green Subscribe button on the landing page switches ON automatically. (Leave it blank and only the free preview shows — that's the current state.)

3 Test a real checkout

Click your own Subscribe button → make sure the Payhip checkout opens and completes. Once a test payment goes through, the gate is fully closed and you're ready to market.
