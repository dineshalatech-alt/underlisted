Go-Live Checklist — Worker + Payment

The exact, in-order steps to fill the app with real homes and switch on payment · updated 2026-06-14

The code is finished and cost-guarded. The refresh worker, its daily schedule, and the Payhip "Subscribe" button are all built and committed. Everything below is account / dashboard clicks — no coding. Do them strictly in order, top to bottom.

The whole picture in one line

Think of the app as a shop. The worker is the stock-clerk who fills the shelves (the database) with real homes. Customers just walk in and read the shelves — they never trigger a costly trip to the supplier. Today we hire the clerk for the first time. We do it small and careful first, check the shelves filled correctly, then open the doors wide.

The checklist (tick these in order)

  • Step 1 — Confirm the RentCast upgrade is active. Log in to RentCast → check your plan shows the new paid tier and your request quota has gone up. Do not continue until this reads "active". Owner
  • Step 2 — Add the GitHub Actions Secrets (full list + clicks in Part A below). Owner
  • Step 3 — Run ONE small controlled worker test (the safe-first-run recipe in Part B). We deliberately keep this first run to a handful of homes so it costs almost nothing.
  • Step 4 — Verify the database filled with real homes (Part C).
  • Step 5 — Open coverage to the full nationwide city list and let the daily schedule take over (Part D).
  • Step 6 — Create the Payhip $18.99 product + paste the link (Part E — your money call). Owner
  • Step 7 — Flip payment ON and test one real checkout (Part E). Owner

Part A — Add the GitHub Actions Secrets

These are like keys kept in a locked drawer. The worker takes them out only while it runs, and they are never written into any file. You paste them once. This is a human step — only you can log in to GitHub and do it.

A1 Open the secrets page

In a browser, go to your repo:
github.com/dineshalatech-alt/underlisted
Then click: Settings (top menu) → in the left sidebar Secrets and variables → Actions → the green New repository secret button.

A2 Add each secret below

For each row: type the Name exactly as shown (capitals, underscores), paste the Value, click Add secret, then repeat for the next one.

Secret name (type exactly)Required?Where you get the value
DATABASE_URLRequiredThe same Supabase connection string the app already uses. It is in your Streamlit Cloud app → Settings → Secrets, on the DATABASE_URL line. Copy that exact value. (It starts with postgresql://postgres.nemvz...)
RENTCAST_API_KEYRequiredYour RentCast dashboard → API key. The same key that's in your local .env file.
RESEND_API_KEYOptionalResend dashboard → API Keys. Only needed for alert emails. Same value as in your .env. Skip for now if you like.
ALERT_FROM_EMAILOptionalThe "from" address for alert emails (e.g. onboarding@resend.dev for now). Skip if skipping Resend.
STREETVIEW_API_KEYOptionalGoogle Street View key. Only used if photo pre-warm is turned on (it is OFF by default), so you can safely skip this.

Minimum to go: just the two Required ones (DATABASE_URL and RENTCAST_API_KEY) are enough for the worker to fill the shop.

Part B — The ONE small controlled worker test

Why small first? The worker normally pulls many cities. The first time, we shrink it to a couple of neighborhoods so it makes only a handful of supplier calls — just enough to prove the pipe works (worker → database → app). Once we see real homes land, we open it up wide (Part D). This costs next to nothing and protects your new quota.

B1 Tell me to set the "small-batch" safe mode

This is my job (a tiny config tweak, no cost). On your word I will, in config/cache.yaml, temporarily set:

max_estimates_per_run: 0  # skip all value/rent billing on the first run

and shrink config/cities.yaml to one city with two ZIPs (e.g. Sacramento 95820, 95823). That makes the first run cost about 2 supplier calls total — the cheapest possible proof. I commit + push it; nothing bills yet.

B2 Run the worker once, by hand

In your repo: Actions tab (top menu) → click "Refresh listings cache" in the left list → click the Run workflow button on the right → confirm Run workflow. It runs on GitHub's computers (not yours) and turns green in a few minutes.

There is also a manual local option (only if you ever want it): .venv\Scripts\python.exe -m worker.refresh_worker in PowerShell — but the GitHub button is the recommended way and needs nothing installed on your laptop.

Part C — Verify the database filled with real homes

C1 Check the Admin / Usage page

Open the live app → open the Admin / Usage page. Confirm:

  • Backend reads PostgreSQL (not SQLite).
  • The listing count is above zero — and shows real addresses, not sample data.
  • The latest worker run shows "ok" with a small billable_calls number (should be tiny on this first small run).

C2 Eyeball Browse Deals

Open Browse Deals and confirm real homes appear for the test city with sensible Deal Scores. If you see real listings here, the whole pipe works. 🎉

Part D — Open coverage to the full USA

D1 Tell me to restore the full city list

Once the small test is verified, on your word I put back the full nationwide city list (~16 metros across TX, OH, IN, MO, TN, AL, FL, GA, NC, PA, OK plus the California ZIPs) and restore max_estimates_per_run. That full daily run makes on the order of 30–130 supplier calls — well within a paid tier. From then on the worker refreshes automatically every day at ~9:00 UTC; you don't touch anything.

Part E — Turn payment ON ($18.99/mo founding)

This whole section is YOUR call. It involves money and selling. I will not create products, change pricing, or flip the button on without you doing these steps yourself.

E1 Create the product on Payhip Owner

Sign in at payhip.com → create a Subscription product → price it $18.99/month (founding price). Payhip handles the card payment and gives you a link like https://payhip.com/b/XXXXX.

E2 Paste the link into one file Owner

Open config/cache.yaml (line 15) and put your link between the quotes:

checkout_url: "https://payhip.com/b/XXXXX"

Save, commit, push. The green Subscribe button switches ON automatically. (Leave it blank and only the free preview shows — that's the current safe state.) I can do the commit/push for you once you've decided — just say so.

E3 Test one real checkout Owner

Click your own Subscribe button → confirm the Payhip checkout opens and completes. Once a test payment goes through, the gate is fully closed and you're ready to market.
