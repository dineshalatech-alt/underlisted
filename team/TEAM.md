## How we improve (the loop)
1. Do the work; verify it really works (run it, read the output).
2. Log it in `../PROGRESS.md` (what we did, current state, next step).
3. When something breaks or a better way appears, fix the tool AND update the relevant doc
   (workflow / agent / `develop/`), so the system gets stronger — not just patched once.
4. Keep `develop/` (goals, map, how-we-win) and this roster current as the project evolves.
