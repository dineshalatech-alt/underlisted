The big idea

Building an app = opening a shop 🏪

One person can't do everything. So you hire specialist helpers — each website is one helper with one job.

Your app is the manager 🧑‍💼 that tells all the helpers what to do.

The key concept

What is a "key"? 🔑

A key is a secret password a helper gives you, so your app can prove "I'm allowed to talk to you."

Think of it like a VIP badge 🪪 — show it, and the helper does the job.

No key = the helper says "I don't know you — go away." That's why nothing works until the key is in place.

Safety

Keys live in a locked box 📦

  • On your computer → the .env file
  • In the cloud → Streamlit "Secrets"
🔒 Golden rule: never paste a key into a chat, email, or anywhere public — it's like shouting your house-key code in the street.

The pattern

Connecting any 2 websites = same 3 steps

  • 1. The helper gives you a key (copy a secret code)
  • 2. You put the key in your secret box (Streamlit Secrets)
  • 3. Your app shows the key every time it talks to that helper
Every connection — RentCast, Resend, the database — works this exact way. 🎯

Special case

The database key is address + password 🗄️

Most keys are just a password. DATABASE_URL also says where the filing cabinet is.

postgresql:// user : password @ address : port / database (who) (secret) (where) (door) (cabinet)

The app reads this note → goes to that cabinet → unlocks it → files the info.

It all ties together

The journey of one customer 🚶

  • Googles "cheap homes" → finds underlistedhomes.com (Namecheap → Netlify) 📋
  • Types email to join → Netlify saves it ✍️
  • Opens the app (Streamlit) 👨‍🍳
  • App pulls deals from Supabase 🗄️ (filled earlier from RentCast 🚚)
  • Great deal appears → Resend emails them 📮
Every arrow is unlocked by a key 🔑.
